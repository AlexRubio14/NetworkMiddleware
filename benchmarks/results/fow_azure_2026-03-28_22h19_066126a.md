# FOW Azure Benchmark — 2026-03-28_22h19 — commit 066126a

**Goal:** Prove VisibilityTracker + SpatialGrid filter cross-group entities
when two groups of bots are placed beyond DEFAULT_VISION_RANGE=150u.

## Spawn Layouts

| Mode    | Zone A                        | Zone B                       | Min distance |
|---------|-------------------------------|------------------------------|--------------|
| cluster | x∈[-80,80] y∈[-80,80]        | same zone                    | 0u           |
| bimodal | x∈[-400,-200] y∈[-100,100]   | x∈[200,400] y∈[-100,100]    | ~400u        |

Vision range = 150u → bimodal groups are invisible to each other.

## Results

| Scenario                     | Connected | Avg Tick | Full Loop | Out          | In           | Delta Eff.   |
|-----------------------------|-----------|---------|---------|--------------|--------------|--------------|
| Cluster (FOW=OFF baseline)   | 40/40     | 0.71ms  | 2.75ms  | 3224.4kbps   | 345.2kbps    | 72%          |
| Bimodal (FOW=ON groups)      | 40/40     | 0.58ms  | 2.28ms  | 2848.0kbps   | 345.0kbps    | 69%          |

## FOW Effectiveness

- Out_bimodal / Out_cluster = **88.3%**
- Bandwidth reduction from FOW = **11.7%**
- Target: ~50%

## Notes

- Server: Docker container with SPAWN_ZONE_MODE env var, same image as production.
- Bots: WSL2 local → Azure Switzerland North over real WAN (no tc netem).
- DEFAULT_VISION_RANGE = 150u; bimodal min cross-group distance ≈ 400u.
- The original 'netserver' container is restored after each scenario.
