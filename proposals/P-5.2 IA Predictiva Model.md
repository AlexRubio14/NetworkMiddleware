# P-5.2 — IA Predictiva: Modelo Concreto

**Fase:** [[DEVELOPMENT_MEMORY#Fase 5|Fase 5]]

## Problema

El roadmap dice "modelo de regresión" sin especificar cuál. Sin concretar, no se puede implementar ni citar académicamente.

## Opciones analizadas

| Modelo | Precisión | Coste CPU | Citable |
|--------|-----------|-----------|---------|
| Regresión lineal | Baja (no funciona para click-to-move con cambios de dirección) | Mínimo | No suficiente |
| **Filtro de Kalman** | Alta para movimiento con ruido gaussiano | Bajo-medio | Kalman (1960) ✓ |
| LSTM | Muy alta | Alto (inaceptable en tiempo real) | Excesivo para TFG |

## Propuesta

**Filtro de Kalman** — académicamente citable, implementable en C++, coste computacional bajo.

## Preguntas a decidir

- [ ] ¿Kalman filter o empezamos más simple (velocidad × tiempo)?
- [ ] ¿Cuándo se activa la predicción? ¿Solo ante packet loss o siempre como suavizado?
- [ ] ¿Cómo medimos el error del 5%? ¿Distancia euclidiana?
