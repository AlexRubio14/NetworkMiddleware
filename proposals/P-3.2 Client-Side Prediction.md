# P-3.2 — Client-Side Prediction

**Fase:** [[DEVELOPMENT_MEMORY#Fase 3|Fase 3]]
**Depende de:** [[P-3.1 ACK Bitmask]]
**Bloquea:** [[P-6.2 Reconciliation Visual]]

## Problema

Sin client-side prediction, 50ms de ping = 50ms de input delay en el click-to-move. Inaceptable para MOBA competitivo.

## Propuesta

El servidor acepta `InputCommands` con timestamp. El cliente aplica su propio input inmediatamente y reconcilia cuando llega el estado autoritario.

**Qué predice el cliente (propuesta):**
- Sí: movimiento propio
- No: daño recibido, estados (stunned, dead) — siempre del servidor

**Qué necesita el servidor:**
- Estructura `InputCommand { sequence_id, timestamp, input_type, input_data }`
- Cola de inputs por RemoteClient procesada en orden
- Respuesta con `sequence_id` del input confirmado
