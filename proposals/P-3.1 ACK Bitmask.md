# P-3.1 — ACK Bitmask en el Header

**Fase:** [[DEVELOPMENT_MEMORY#Fase 3|Fase 3]]
**Bloquea:** [[P-3.2 Client-Side Prediction]], [[P-6.1 Entity Interpolation]]

## Problema

El diseño actual del header (Sequence, Ack, Type) usa ACK por paquete individual. Esto es lento para avanzar el baseline.

## Propuesta

Añadir un campo `AckBits` (uint32) al header que confirme los 32 paquetes anteriores de una vez. Añadir también `Timestamp` (uint32) necesario para la interpolación en Fase 6.

```
[ Sequence (16b) | Ack (16b) | AckBits (32b) | Type (8b) | Timestamp (32b) ]
```

**Ventaja:** El baseline avanza 32x más rápido.
**Coste:** 4 bytes extra por paquete.
**Referente:** Valorant, Fiedler (2016).

- [ ] ¿Añadimos Timestamp al header ahora o en Fase 6?
