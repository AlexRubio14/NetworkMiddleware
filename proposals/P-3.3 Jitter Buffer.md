# P-3.3 — Jitter Buffer

**Fase:** [[DEVELOPMENT_MEMORY#Fase 3|Fase 3]]
**Depende de:** [[P-3.1 ACK Bitmask]] (necesita Timestamp en header)

## Problema

Clock Synchronization (3.4) menciona RTT pero no especifica el jitter buffer. Sin él, los paquetes que llegan irregulares causan stutter visual aunque la latencia media sea buena.

## Propuesta

El cliente bufferiza 2-3 ticks antes de renderizar. El mundo se ve ligeramente en el pasado (~20-30ms) pero el movimiento es perfectamente fluido.

**Tradeoff:**
- Buffer pequeño → posible stutter
- Buffer grande → latencia percibida mayor
